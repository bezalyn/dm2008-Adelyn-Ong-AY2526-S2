When i was a kid, I often find myself observing the little rain droplets that flow pass the car window, hoping that the droplet i picked would collide with its other fated half( another random droplet i picked). 
It was pretty straightforward and I knew that i had to start with a class that determines how my droplet looked like, I knew that because it was similar to our previous assignement, flappy bird. Create a class, then call the pipes out from the array using for loop, then you can have many of them, different variations of them.

The droplet is made primarily with 3 balls, a black for shadow, a grey for body and a white for highlights.
For its movement, I added a variable that had a random range, and added this variable to the droplet's X position, so it would wiggle.
With the background alpha turned down, we see a wiggly trail of the rain droplets, similar to how rain flows down a surface.
I also added a wind element in a variable, so that all droplets gravitates to a certain direction.

After the visuals are completed, i needed to add collision, this part was abit complex for me because this time i need one collding droplet to dissappear and one to increase in its size based of the one that dissappeared. I asked Chat to help with that.

For the interactivity i attached mouse X and mouse Y to my main droplet positiom, so that i have a droplet that follows the mouse wherever it moves.
I also needed a collision for this, and i needed whatever that hit my mouse droplet to reset back on top, while my mouse droplet grows in size.
The logic is that if droplet collides with mouse droplet, droplet resets position and mouse droplet increases in g variable.
